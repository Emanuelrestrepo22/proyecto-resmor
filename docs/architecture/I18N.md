# Arquitectura i18n — Resmor Web

**Agente:** AGT-03 Documentation
**Estado:** Implementado · 2026-05-17
**Idiomas soportados:** Español (default) · Portugués (pt-BR) · Inglés (en-US)

---

## 1. Visión general

Sistema **i18n client-side puro**, sin framework. JavaScript vanilla aplica un diccionario JSON por idioma sobre los elementos HTML decorados con atributos `data-i18n*`. Sin reload de página, sin SSR, sin dependencias externas. Compatible con el stack actual (HTML + Bootstrap CDN + SCSS).

### Por qué este enfoque

| Criterio                  | Decisión                                                      |
| ------------------------- | ------------------------------------------------------------- |
| Stack actual              | HTML estático + Bootstrap CDN — sin Node runtime en producción |
| Mantenibilidad            | Diccionarios JSON simples, fáciles de editar manualmente      |
| Performance               | Un solo fetch por idioma, cacheado por el browser             |
| SEO                       | Acepta `?lang=` URL param para indexación dirigida            |
| Accesibilidad             | Actualiza `<html lang="">` y `og:locale` dinámicamente        |
| Persistencia              | localStorage + auto-detección con `navigator.language`        |

### Lo que **no** hace (intencional)

- No genera URLs separadas por idioma (`/es/`, `/pt/`, `/en/`). Una sola URL canónica con override por query string.
- No traduce contenido del backend (no hay backend).
- No re-renderiza React/Vue (no hay framework).

## 3. Convenciones de atributos HTML

Cualquier elemento del sitio puede internacionalizarse usando uno o más de estos atributos:

| Atributo                | Comportamiento                                                        | Ejemplo                                                  |
| ----------------------- | --------------------------------------------------------------------- | -------------------------------------------------------- |
| `data-i18n="key.path"`  | Reemplaza `textContent` del elemento                                   | `<h1 data-i18n="hero.indexTitle">Fallback</h1>`         |
| `data-i18n-html="..."`  | Reemplaza `innerHTML` (permite HTML embebido como `<strong>`)         | `<p data-i18n-html="about.introP1">...</p>`             |
| `data-i18n-attr="..."`  | Setea atributos. Formato `attr:key[,attr:key]`                        | `<input data-i18n-attr="placeholder:contact.formNamePlaceholder,aria-label:contact.formNameLabel">` |
| `data-i18n-wa="key"`    | Reemplaza el `href` con `wa.me/<num>?text=<wa.key>`                   | `<a data-i18n-wa="airport">...</a>`                     |
| `data-set-lang="es"`    | Botón que dispara `setLang('es')` al click                            | `<button data-set-lang="pt">PT</button>`                |
| `data-lang-switcher`    | Marker para el contenedor del switcher (estilos del estado activo)   | `<div data-lang-switcher>...</div>`                     |
| `data-page-title-key`   | En `<html>`, indica qué key del diccionario usar como `<title>`       | `<html data-page-title-key="hero.indexTitle">`          |

**Fallback:** el texto plano dentro del elemento sirve como fallback si la clave no existe en el JSON o si el fetch falla.

## 4. Convenciones del diccionario JSON

### Estructura

Cada `js/i18n/<lang>.json` es un objeto plano con secciones por dominio:

```json
{
  "meta":            { "lang": "es", "flag": "🇦🇷", "label": "Español", "shortLabel": "ES" },
  "html":            { "lang": "es", "ogLocale": "es_AR" },
  "topbar":          { ... },
  "nav":             { ... },
  "hero":            { "indexTitle": "...", "aboutTitle": "...", ... },
  "porQue":          { ... },
  "servicesSection": { ... },
  "pasos":           { ... },
  "ctaClose":        { ... },
  "footer":          { ... },
  "about":           { ... },
  "pricing":         { ... },
  "contact":         { ... },
  "gallery":         { ... },
  "wa":              { "general": "Hola Resmor, quiero cotizar", "airport": "...", ... }
}
```

### Reglas para agregar/editar keys

1. **Identidad de keys es estable.** Cambiar `hero.indexTitle` rompe el HTML — sólo se renombra cuando se hace replace coordinado en los 3 JSON + HTML.
2. **Las 3 versiones deben mantener la misma estructura.** Si en `es.json` agrego `foo.bar`, debe existir en `pt.json` y `en.json` (aunque sea con el mismo texto temporal).
3. **HTML válido en valores `data-i18n-html`.** El motor inyecta sin sanitizar — sólo permitir HTML que vos controlás.
4. **WhatsApp messages crudos** (sin URL-encoding). El motor encoda con `encodeURIComponent`.

## 5. Detección de idioma

Orden de precedencia (primero gana):

```text
1. ?lang=<es|pt|en>     (URL param)
2. localStorage.resmor.lang
3. navigator.language[0..2]  // 'pt-BR' → 'pt', 'en-US' → 'en'
4. DEFAULT_LANG = 'es'
```

La elección queda persistida en `localStorage.resmor.lang` para futuras visitas.

## 6. API pública

El motor expone `window.ResmorI18n` con:

```js
ResmorI18n.setLang('pt')                   // cambia idioma (async)
ResmorI18n.getLang()                        // → 'es' | 'pt' | 'en'
ResmorI18n.getDict()                        // → objeto del diccionario actual
ResmorI18n.t('hero.indexTitle')             // → string traducido
ResmorI18n.supported                        // → ['es', 'pt', 'en']
```

### Evento personalizado

Cada cambio de idioma dispara:

```js
document.dispatchEvent(new CustomEvent('resmor:langchange', {
  detail: { lang, dict }
}));
```

Usalo para reaccionar a cambios (ej: re-renderizar widgets externos).

## 7. Flujo de carga

```text
DOMContentLoaded
   └─ attachSwitcher()                       // bind click en botones data-set-lang
   └─ detectInitialLang()                    // URL → localStorage → navigator → default
   └─ setLang(lang)
       ├─ loadDict(lang)                     // fetch ./js/i18n/<lang>.json (cacheable)
       ├─ applyText / applyHtml / applyAttrs / applyWaLinks
       ├─ applyHtmlMeta                      // <html lang>, og:locale, <title>
       ├─ updateSwitcherUI                   // marca el botón activo
       └─ dispatchEvent('resmor:langchange')
```

## 8. Cómo agregar un texto nuevo a una página

Workflow recomendado:

1. **HTML:** agregar el elemento con `data-i18n="<seccion>.<nombre>"` y un fallback en español.
   ```html
   <p data-i18n="hero.newTagline">Texto en ES como fallback</p>
   ```
2. **es.json:** agregar la key.
3. **pt.json:** agregar la key con la traducción al portugués.
4. **en.json:** agregar la key con la traducción al inglés.
5. **Recargar el sitio** (no requiere recompilar SCSS ni nada).

---

## 9. Cómo agregar un idioma nuevo (ej: italiano)

1. Crear `js/i18n/it.json` con la misma estructura que `es.json`.
2. En `js/i18n.js`:
   - Agregar `'it'` al array `SUPPORTED`.
3. En cada HTML, agregar el botón al switcher:
   ```html
   <button class="lang-switcher__btn" data-set-lang="it" type="button" aria-pressed="false">
     <span class="flag">🇮🇹</span>IT
   </button>
   ```
   - O ejecutar `scripts/apply_i18n.py` actualizando la lista del switcher.
4. (Opcional) Agregar variante en `_11lang-switcher.scss` si querés un color distinto.

## 10. Internacionalización de WhatsApp

El motor reescribe el `href` de cualquier elemento con `data-i18n-wa="key"` así:

```text
https://wa.me/5491123048846?text=<encodeURIComponent(dict.wa.<key>)>
```

El JSON guarda mensajes en **texto plano** (sin encoding):

```json
"wa": {
  "general": "Hola Resmor, quiero cotizar",
  "airport": "Necesito traslado al aeropuerto",
  "executive": "Necesito transporte ejecutivo"
}
```

Cuando el usuario en EN clickea "Get a quote", abre WhatsApp con `Hi Resmor, I'd like a quote`. El equipo de Resmor recibe el mensaje en el idioma del cliente → indicador útil para responder en el idioma correcto.

## 11. SEO

### Sitemap (recomendado a futuro)

URL canónica única por página + variantes `?lang=` con `<link rel="alternate" hreflang="...">`:

```html
<link rel="alternate" hreflang="es" href="https://resmor.com.ar/index.html">
<link rel="alternate" hreflang="pt" href="https://resmor.com.ar/index.html?lang=pt">
<link rel="alternate" hreflang="en" href="https://resmor.com.ar/index.html?lang=en">
<link rel="alternate" hreflang="x-default" href="https://resmor.com.ar/index.html">
```

**Pendiente:** agregar estos `<link rel="alternate">` cuando se confirme dominio final.

### Indexación por buscadores

Como el contenido se renderiza client-side, Googlebot lee el JSON y aplica el idioma. La práctica probada:

- Mantener fallback en ES en el HTML estático (lo que ve un crawler sin JS = español).
- Para indexación dirigida en EN/PT, podemos crear landing pages estáticas separadas (`/airport-transfer-buenos-aires.html`, `/transfer-aeroporto-buenos-aires.html`) **fuera** de este sistema, con copy estático en el idioma destino.

## 12. Limitaciones conocidas

| Limitación                                       | Mitigación                                  |
| ------------------------------------------------ | ------------------------------------------- |
| Flash of untranslated content (FOUC)             | Fallback HTML en ES — minimiza el efecto    |
| El JSON inyecta HTML sin sanitizar               | Sólo permitir HTML controlado por el equipo |
| Sin pluralización ni interpolación de variables  | Se puede agregar a futuro si se necesita    |
| `localStorage` puede estar deshabilitado         | El sitio sigue funcionando, no persiste     |
